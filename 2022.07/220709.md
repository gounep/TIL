## class vs. object

class는 조금 더 연관있는 데이터를 한데 묶어 놓는 container 같은 아이

```javascript
class Person{
    name; // 속성(field)
    age; // 속성(field)
    speak(); // 행동(method)
}
```

class => fields + methods

fields만 들어 있는 경우 -> `data class`

관련있는 변수나 함수들을 묶어 놓은 것 -> `class`

class 안에서도 내부적으로 보여지는 변수와, 밖에서 보일 수 있는 변수를 나눠서 `encapsulation` (캡슐화), 상속, 다형성

=> 객체 지향 언어

---

`class` -> 붕어빵을 만들 수 있는 틀 (틀만, 템플릿만 정의)

- template
- declare once 
- no data in

`object` -> 클래스를 이용해서 실제로 데이터를 넣어서 만드는 것

- instance of a class (클래스를 이용해서 새로운 instance를 생성하면 object가 됨)
- created many times
- data in

=> class는 정의만 한 것이라서 실제로 메모리에 올라가지 않지만, 이렇게 실제로 data를 넣으면 object는 메모리에도 올라간다

그래서 붕어빵에 무얼 넣느냐에 따라 팥붕어빵, 피자붕어빵이 되는 것처럼 만들어진 붕어빵 자체는 object, 붕어빵 틀은 class
