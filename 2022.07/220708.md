JS 도 procedural language 중 하나라고 볼 수 있어, function 이 굉장히 중요한 역할을 담당한다.

따라서, 때로는 sub-program이라고 부른다. 프로그램 안에서 또 각각의 작은 단위의 기능들을 수행하는 것이 바로 function이다.



어떤 언어를 배우든 function은 대체적으로 input(parameters)을 받아서 이것들을 잘 처리한 다음에 output을 return한다.



언어 자체에 존재하는 function들을 쓸 때, 또는 API들을 쓸 때 함수의 이름을 보고 어떤 일을 하겠구나 생각할 수 있고, 전달해야 할 파라미터들이 무엇인지, 어떤 값이 리턴되기를 기대할 수 있는지 인터페이스를 보고서 예상할 수 있다.

그래서 함수는 input, output이 중요하고 function의 이름을 잘 정하는 것이 중요하다.



- 클론코딩 ZOOM 책으로 1회독 -> 웹소켓, socket.io, WebRTC 개념을 대략적으로나마 이해해볼 수 있었다.
  - 웹소켓 : HTTP와 다른 방식으로 동작하는 프로토콜. 마치 서로 악수를 하는 것처럼 사용자와 서버를 연결해준다. 이런 악수가 한 번 성립되면 서로 간에 연결이 이루어진 상태가 됨. 이 상태에서 서버는 사용자가 누구인지도 알고 있고, 원한다면 서버가 사용자에게 메시지 보내는 것도 가능함. 이렇게 서로에게 바로 갈 수 있는 양방향 연결이 되어 있어서 웹소켓을 이용하면 실시간 기능을 얼마든지 구현 가능함.
  - socket.io : 등장한 지 오래되어 안정적이고 편리한 기능을 무척 많이 제공하는 놀라운 라이브러리. 실시간 기능을 구현할 수 있고, 이벤트를 기반으로 한 양방향 통신도 할 수 있다. (실시간, 양방향, 이벤트에 기반한 통신 등을 모두 지원하는 라이브러리) '웹소켓을 활용하는 라이브러리'
  - webRTC : web real-time communication. 우리가 peer-to-peer로 데이터 교환을 할 수 있게 해준다. `peer-to-peer`란 인터넷에 연결된 다수의 개별 사용자들이 중개 기관을 거치지 않고 데이터를 직접 주고 받는 것. 보내고 싶은 메시지나 영상을 서버를 거치지 않고 상대방에게 직접 전달할 수 있다. 다만, 서버가 없어도 된다는 말이 아니다. `signaling`을 하기위해 서버를 사용해야 한다. 시그널링은 사용자들을 서로 연결하는 작업이다.(브라우저 간의 연결)
